# HireInsight 文档集

> 把 `FindJobs-Agent`（应聘者视角）翻转为 **HR / 企业管理者视角**的 AI 招聘管理系统
> v1.0 ｜ 2026-06-13 ｜ 状态：待评审

---

## 一句话

HR 批量上传简历后，系统复用并**反转**原仓库的 AI 引擎，自动完成解析、打标、画像、为 JD 荐人、AI 预筛面试；管理者用 **BI 看板**实时监控每位 HR 专员的工作量与效能。

**核心比喻**：应聘者用它"被评估"，招聘者用它"做评估"——AI 引擎不变，评估的主客体对调。

---

## 三个必备功能（已落到代码）

1. **批量简历 + AI 全流程闭环** — 扩展 `/api/resume/upload` + 异步队列，串起 `resume_parser`→`tag_rate`→画像→匹配→`AI_interviewer` 预筛。
2. **BI 看板监控 HR 专员（难点）** — 原仓库零埋点，靠新建 `events` 表自产数据，做漏斗/效能/瓶颈三层下钻。
3. **复用改造原仓库** — 逐文件给了复用度与改造动作。

## 改造逻辑速览

```
原 FindJobs-Agent（应聘者）        →   HireInsight（招聘者）
─────────────────────────────────────────────────────────
resume_parser 解析我的简历        →   批量解析候选人简历池（+异步）
tag_rate 给我技能打分             →   给候选人打标评分入库（近乎直接复用）
爬岗位推荐给我                    →   弃爬虫；录 JD→从简历池荐人（匹配反转）
AI_interviewer 帮我练面试         →   AI 预筛候选人、出评估报告（评判反转）
我看岗位列表                      →   HR 看流水线 Kanban + 管理者看 BI（新增）
```
