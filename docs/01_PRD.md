## 1. 产品定位

把 `FindJobs-Agent` 的 AI 引擎从"应聘者被评估"翻转为"招聘者做评估"。HR 批量导入简历，系统自动解析、打标、画像、为 JD 荐人、AI 预筛面试；管理者用 BI 看板监控团队效能。

---

## 2. 角色与权限

| 角色 | 能做什么 | 数据可见范围 |
|---|---|---|
| HR 专员 | 传简历、看候选人画像、推进流水线、发起 AI 预筛 | 仅自己负责的候选人/岗位 |
| 招聘主管/HRD | 分配岗位、看 BI 看板、考核效能 | 全团队 |
| 用人部门面试官 | 看推荐简历、填面试评价 | 被指派的候选人 |
| 系统管理员 | 账号/权限、标签体系、模型配置 | 系统级 |

---

## 3. 功能模块与对应代码

### 模块 A：批量简历导入与处理 ★必备1

| 子功能 | 复用/新增 | 落到代码 |
|---|---|---|
| A1 批量上传（多文件/zip） | 扩展 | 改造 `POST /api/resume/upload` 收多文件；新增任务队列 |
| A2 解析结构化 | 复用 | `resume_parser.py` 包装 `parse_batch()`，输出统一 JSON |
| A3 打标评分 | 复用 | `tag_rate.py`，标签写 `candidate_tags`，兼容 `Python , 5 , AI` 格式 |
| A4 候选人画像 | 新增 | 聚合结构化字段+标签→雷达图+风险提示 |

**用户故事**：作为 HR 专员，我拖拽上传一个含 80 份简历的 zip，系统异步解析并在列表里逐条显示"解析中/已打标/失败可重试"，完成后每份都有技能标签和评分。

**验收**：单批 ≥100 份；关键字段抽取准确率 ≥90%；失败项可单独重试；PII 字段脱敏存储。

### 模块 B：岗位管理与人岗匹配 ★必备1

| 子功能 | 复用/新增 | 落到代码 |
|---|---|---|
| B1 JD 录入与结构化 | 复用 | `job_agent.py` 抽取学历/专业/技能要求 |
| B2 岗找人匹配（反转） | 反转 | 输入 JD 画像，遍历简历池算匹配度，输出排序候选人+理由 |
| B3 智能荐人 | 新增 | 新 JD 发布自动从存量简历池召回（激活沉睡简历） |

**用户故事**：作为 HR，我录入一条"高级后端"JD，系统结构化出要求后，从现有简历池返回按匹配度排序的候选人列表，每个标注"为什么匹配/缺什么"。

**验收**：给定 JD 返回 Top-N 候选人 + 可解释匹配理由。

### 模块 C：AI 预筛面试（反转）★必备1

复用 `AI_interviewer.py`，把"帮候选人练面试给建议"反转为"对候选人作答评分"。

| 接口 | 原用途 | 反转后 |
|---|---|---|
| `/api/interview/start` | 候选人开始练习 | HR 对某候选人发起预筛，按 JD 出题 |
| `/api/interview/answer` | 候选人提交答案得建议 | 候选人异步作答，AI 评分 |

**输出**：预筛报告 = 分项评分 + 亮点 + 疑点（识别"包装简历"）+ 是否建议进真人面试。

**验收**：候选人完成异步作答后，HR 收到结构化报告。

### 模块 D：招聘流水线 ★新增

Kanban：`待初筛 → AI预筛 → 部门面试 → Offer → 入职 / 淘汰`。每次流转写 `events` 表（供 BI）。

**用户故事**：作为 HR，我把通过 AI 预筛的候选人卡片拖到"部门面试"，系统记录流转时间与操作人。

### 模块 E：BI 监控看板 ★必备2（难点，详见 [03](./03_BI看板设计.md)）

管理者看团队漏斗、各专员效能对比、瓶颈环节，可三层下钻。

### 模块 F：系统支撑 ★新增

- F1 RBAC（角色权限隔离）
- F2 操作埋点（写 `events`，BI 数据源）
- F3 PII 脱敏 + 访问审计 + 候选人授权告知

## 4. 优先级（MoSCoW）

| 级别 | 功能 |
|---|---|
| Must | A1-A4、B1-B2、C、D、E1-E2、F1-F2 |
| Should | B3、E3、F3 |
| Could | 自定义标签体系、面试题库管理 |
| Won't（本期） | 外部爬虫、Offer 审批流、薪酬、移动端、实时视频面试 |

---

## 5. 端到端验收场景

1. HR 上传 100 份简历 → 异步完成解析+打标+画像。
2. 录入 JD → 返回排序候选人列表 + 匹配理由。
3. 候选人完成 AI 预筛 → HR 收到评估报告。
4. 候选人在 Kanban 推进 → 全程留痕。
5. 管理者在 BI 看板看各专员漏斗与效能并下钻。
6. 专员只能看自己候选人；PII 脱敏、审计日志可查。
