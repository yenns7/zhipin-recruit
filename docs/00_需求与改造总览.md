# 00 需求与改造总览 — HireInsight

> 把 `FindJobs-Agent`（应聘者视角）翻转为 HR / 企业管理者视角的招聘管理系统
> v1.0 ｜ 2026-06-13

---

## 1. 两份来源，各取什么

**来源 A — Moka 文章（理念）**：传统 ATS 是"人操作工具，系统只记录"；AI 原生招聘是"AI 主动推进流程"。Moka 把这条理念拆成可落地的能力链：
> 多格式简历解析成统一格式 + 自动打标签 → 候选人 360° 画像（含隐藏特征/风险）→ 人岗双向匹配与智能推荐 → AI 面试（千人千问、智能追问、多模态测评）→ 全流程数据看板监控效率。

**来源 B — FindJobs-Agent（代码底座）**：一个跑得起来的全栈系统，已实现上面能力链的"应聘者半边"。逐文件清点（来自仓库 README）：

| 文件 | 现在做什么（应聘者视角） |
|---|---|
| `resume_parser.py` | 解析 PDF/Word 简历 → 结构化 |
| `tag_rate.py` | 技能标签打分（1-5），兼容 `Python , 5 , AI` / `Python: 5` 等格式 |
| `job_agent.py` | LLM 抽取 JD 的学历/专业/技能要求、岗位分类 |
| `tech_taxonomy.json` / `all_labels.csv` | 岗位分类体系 + 技能标签库 |
| `AI_interviewer.py` | 按 JD 生成面试题、多轮对话、给候选人反馈 |
| `pipeline.py` | 爬取→分析→评分→展示 四段流水线 |
| `api_server.py` | Flask，已有 `/api/jobs`、`/api/resume/upload`、`/api/resume/analyze`、`/api/interview/start`、`/api/interview/answer` |
| `job_crawler_v2.py` / `_selenium.py` | 爬腾讯/网易/字节等公开岗位 |
| `FrontEnd/`(React+TS) | `JobsPage` / `ResumePage` / `InterviewPage` |

## 2. 核心洞察：方向反转，不是重写

原系统是"**一个人**找工作"，目标系统是"**一个组织**招人"。同一套 AI 能力，作用方向反转即可复用大部分：

| 原能力 | 反转后 | 复用判断 |
|---|---|---|
| 解析**我的**一份简历 | 批量解析**候选人**简历池 | `resume_parser.py` 核心可用，需加批量+异步 |
| 给**我的**技能打分 | 给**候选人**打标评分入库 | `tag_rate.py` 几乎直接用 |
| 爬岗位→**推荐给我** | 弃爬虫，HR 录 JD→**从简历池荐人** | 匹配逻辑反转 input/output |
| 帮**我**练面试、给我建议 | **AI 预筛候选人**、出评估报告 | `AI_interviewer.py` 把"教学"改"评判" |
| **我**看岗位列表 | **HR** 看候选人流水线 + **管理者**看 BI | 全新 |

> 一句话：**应聘者用它"被评估"，招聘者用它"做评估"。** AI 引擎不变，评估的主客体对调。

## 3. 三个必备功能（你点名的）落到代码

**必备 1 — 批量简历 + AI 全流程闭环**
- 入口：扩展 `/api/resume/upload` 支持多文件/zip，新增异步任务队列。
- 链路：`resume_parser` 批量解析 → `tag_rate` 打标 → 新增"候选人画像" → `job_agent` 已结构化的 JD 做匹配 → 进流水线 → `AI_interviewer` 预筛。
- 闭环 = 简历进系统后无需人工搬运，自动走完识别/打标/画像/匹配/预筛。

**必备 2 — BI 看板监控 HR 专员（难点）**
- 原仓库零埋点，数据要自产：每个 API 操作落 `events` 表。
- 看板让管理者看到每个专员的处理量、转化率、瓶颈。详见 `03`。

**必备 3 — 复用改造原仓库**
- 复用度排序：`tag_rate`/`taxonomy`（高）→ `resume_parser`/`job_agent`/`AI_interviewer`（中，需反转/批量）→ `api_server`/前端（扩展）→ 爬虫（弃用）。

## 4. 边界（待你确认的 6 条假设）

| # | 假设 | 不确认会怎样 |
|---|---|---|
| A1 | 简历靠 HR 导入，**不爬外部网站** → 弃 `job_crawler` | 决定爬虫去留 |
| A2 | LLM 沿用 OpenAI（`API_key.md`），可换国产模型 | 影响合规/成本 |
| A3 | 单企业私有化部署，万级简历/年，存储升级 CSV/JSON → PostgreSQL | 影响选型 |
| A4 | BI 数据 = 系统操作埋点，不接考勤/IM | 决定埋点量 |
| A5 | AI 面试是**异步预筛**（候选人自助作答），非实时视频 | 决定是否上多模态 |
| A6 | 数据不出企业，含 PII，需脱敏+审计 | 合规红线 |

> 未确认前，后续 `01`–`04` 按上述假设推进并就地标注。
